# AI 图片生成工具 — 设计方案

## 1. 项目概述

基于 Flutter 构建一个跨平台（iOS/Android/Web/Desktop）的 AI 图片生成与编辑工具，封装两个核心 API：

| 接口 | API ID | 方法 | 用途 |
|------|--------|------|------|
| `POST https://jeniya.cn/v1/images/generations` | `api-448580046` | 文生图 | 根据文本提示生成图片 |
| `POST https://jeniya.cn/v1/images/edits` | `api-448580045` | 图编辑 | 基于已有图片+文本提示进行编辑 |

---

## 2. 页面结构

整体采用 **底部导航栏（BottomNavigationBar）** 切换两个主要页面：

```
App

 │    └── Tab 1: 图编辑（EditPage）

 │    ├── 参数配置面板

      ├── 参数配置面板

      └── 结果展示区（编辑后的图片 + 下载）
```

## 3. 各页面详细设计

### 3.1 文生图页面（GeneratePage）

#### 功能描述
输入文字描述，AI 根据描述生成对应图片。

#### UI 布局（从上到下）

| 区域 | 组件 | 说明 |
|------|------|------|
| 提示词输入 | `TextField`（多行） | 必填，最大 1000 字符，带字数统计和清空按钮 |
| 参数配置 | 模型选择 | 下拉菜单：`gpt-image-2`（默认）、可扩展其他模型 |
| | 画布尺寸选择器 | 支持：`1024x1024`（方）、`1536x1024`（横）、`1024x1536`（竖）、`2048x2048`（2K方）、`2048x1152`（2K横）、`3840x2160`（4K横）、`2160x3840`（4K竖）、`auto` |
| | 图片格式 | 下拉选择：`png`、`jpeg`、`webp` |
| | 画质选择 | 下拉选择：`low`、`medium`、`high`、`auto` |
| | 生成数量选择器 | Stepper/slider，范围 1~10 |
| 操作按钮 | `ElevatedButton` | "生成图片"按钮，带 loading 状态 |
| 结果展示 | `GridView` | 展示生成的图片列表，支持点击预览大图、保存到本地 |

#### 页面状态设计

| 状态 | 展示方式 |
|------|----------|
| **空状态** | 占位图 + 引导文案（"输入描述，开始生成你的 AI 图片"） |
| **加载中** | 生成按钮禁用 + 骨架屏占位 + 顶部进度提示条（"AI 正在生成中，请稍候..."） + 取消按钮 |
| **成功** | GridView 图片网格 + 每张图支持点击预览 |
| **错误** | SnackBar 顶部提示错误信息 + 保留上次成功结果（如果有） |
| **部分失败** | 成功的图片正常展示 + 失败位置显示重试按钮 |

#### API 调用

```dart
POST https://jeniya.cn/v1/images/generations
Headers:
  Content-Type: application/json
  Authorization: Bearer {{API_KEY}}

Body (JSON):
{
  "model": "gpt-image-2",
  "prompt": "A childrens book drawing of a veterinarian...",
  "n": 1,
  "size": "1024x1024",
  "quality": "low",
  "format": "jpeg"
}
```

> 尺寸规则：长边 ≤ 3840px，宽高均为 16px 倍数，长宽比 ≤ 3:1，总像素 655360 ~ 8294400

#### 响应处理

> **重要：gpt-image-2 始终返回 base64 编码的图像**，不是 URL。需要解码后使用 `MemoryImage` 展示。

**响应结构适配：**
- 文档给出的示例是 `chat.completion` 模板（可能是文档复制错误），实际图片 API 响应需按 `images` 结构适配
- 适配策略：优先解析 `data[]` 数组（OpenAI 标准格式），每个元素含 `b64_json` 字段；如果没有 `data` 字段再尝试从 `choices[].message.content` 中提取

- 成功：解码 base64 → 内存图片对象 → 存入结果列表 → GridView 展示
- 失败：展示错误提示（如 API Key 无效、请求超时、内容违规等）

### 3.2 图编辑页面（EditPage）

#### 功能描述
上传一张或多张图片，输入编辑描述，AI 根据描述对图片进行编辑修改（支持多图合并等操作）。

#### UI 布局（从上到下）

| 区域 | 组件 | 说明 |
|------|------|------|
| 图片上传 | 图片列表预览 + "添加图片"按钮 | 支持选取多张图片，可拖动排序、移除 |
| 遮罩图片（可选） | 遮罩预览 + "选择遮罩"按钮 | 可选，指定编辑区域（仅对第一张图生效） |
| 提示词输入 | `TextField`（多行） | 必填，gpt-image-2 最大 32000 字符 |
| 参数配置 | 模型选择 | 下拉菜单：`gpt-image-2`、`gpt-image-2-all`、`gpt-image-1`、`flux-kontext-pro`、`flux-kontext-max` 等 |
| | 画布尺寸 | 同文生图尺寸列表 |
| | 画质选择 | `low`、`medium`、`high`、`auto` |
| | 背景透明度 | 可选：`transparent`、`opaque`、`auto` |
| | 内容过滤级别 | 可选：`low`、`auto` |
| | 生成数量 | 范围 1~10 |
| 操作按钮 | `ElevatedButton` | "编辑图片"按钮，带 loading 状态 |
| 结果展示 | `GridView` | 展示编辑后的图片列表 |

#### 页面状态设计

同文生图页面状态设计（空状态 / 加载中 / 成功 / 错误），区别在于：
- 空状态提示："选择一张或多张图片，开始 AI 编辑"
- 加载文案："AI 正在编辑图片，请稍候..."

#### API 调用

```dart
POST https://jeniya.cn/v1/images/edits
Headers:
  Accept: application/json
  Authorization: Bearer {{API_KEY}}

Body (multipart/form-data):
  image: <file[]>    // 必填，支持多张图片（数组形式）
  prompt: "将他们合并在一个图片里面"  // 必填，编辑描述
  mask: <file>       // 可选，遮罩图片
  model: "gpt-image-2-all"  // 可选，模型
  n: "1"             // 可选，生成数量
  size: "1024x1536"  // 可选，尺寸
  quality: "high"    // 可选，画质
  background: "transparent"  // 可选，背景透明度
  moderation: "low"  // 可选，内容过滤级别
```

#### 响应处理
同文生图，gpt-image-2 返回 base64 编码图像。需解码后使用 `MemoryImage` 展示。
- 成功：解析 base64 图片数据 → 存入结果列表 → GridView 展示
- 失败：展示具体错误信息

## 4. 公共模块设计

### 4.1 API 服务层 + Repository 层

采用 **Service + Repository** 双层架构：

```
lib/
 ├── services/
 │    ├── api_client.dart         // HTTP 客户端封装（dio）
 │    └── image_service.dart      // 图片 API 调用（原始请求/响应）
 └── repositories/
      └── image_repository.dart   // 数据转换 + 错误封装 + 缓存策略
```

#### api_client.dart
- 基于 `dio` 封装
- 统一处理 Header（`Authorization`、`Content-Type`）
- 全局错误拦截器（网络错误 → 统一异常类型）
- 支持请求/响应日志（debug 模式）
- 超时配置：连接超时 30s，接收超时 120s（AI 生成耗时较长）

#### image_service.dart
- `generateImage({model, prompt, n, size, format, quality})` → 调用 generations API（JSON）
- `editImage({images, prompt, mask, model, n, size, quality, background, moderation})` → 调用 edits API（multipart/form-data）
- 只负责原始 HTTP 调用和基础解析

#### image_repository.dart
- **数据适配**：将 Service 层原始响应转换为业务模型（base64 解码 → Uint8List → MemoryImage）
- **响应结构容错**：优先解析 `data[].b64_json`，兜底解析 `choices[].message.content`
- **统一错误封装**：网络异常 / API 业务错误 / 参数验证错误 → 统一 `AppException`
- **内存缓存**：LRU 缓存最近生成的图片（最多 N 张，防止 OOM）
- **取消请求**：支持取消正在进行的生成请求（dio CancelToken）

### 4.2 状态管理

使用 **flutter_riverpod** 管理状态：

```
lib/
 └── providers/
      ├── settings_provider.dart   // API Key、主题等全局设置
      ├── generate_provider.dart   // 文生图页面状态
      └── edit_provider.dart       // 图编辑页面状态
```

### 4.3 设置管理

- API Key 配置页（持久化存储，使用 `shared_preferences`）
- 支持自定义 Base URL（方便切换环境）
- 入口：AppBar 右上角齿轮图标（点击跳转设置页）

### 4.4 图片保存功能

- 生成/编辑后的图片支持点击预览大图
- **移动端**：保存到系统相册（使用 `image_gallery_saver`）
- **桌面端/Web**：保存到本地文件（使用 `file_picker` 选择保存路径 / Web 端触发下载）

### 4.6 大图预览页

点击 GridView 中的图片进入全屏预览页：

| 功能 | 说明 |
|------|------|
| 图片缩放 | 双指缩放 / 双击放大缩小 |
| 保存按钮 | 保存到相册（移动端）或本地文件（桌面/Web） |
| 分享按钮 | 调用系统分享（可选，移动端优先） |
| 关闭 | 左滑/右滑关闭、点击关闭按钮 |
| 左右切换 | 多张结果时可左右滑动切换 |

### 4.7 参数面板交互设计

由于参数项较多（模型、尺寸、格式、画质、数量等），采用**可折叠参数卡**：
- 默认折叠高级参数，只显示最常用的（尺寸、数量）
- 点击"更多设置"展开模型、格式、画质等
- 折叠状态用 `ExpansionTile` 或自定义折叠卡
- 展开/折叠状态持久化（记忆用户偏好）

### 4.8 设置页面

**入口：** AppBar 右上角齿轮图标，点击后 `Navigator.push` 进入设置页。

| 设置项 | 组件 | 说明 |
|--------|------|------|
| API Key | `TextField`（密码模式） | 必填，保存到本地存储 |
| Base URL | `TextField` | 可选，默认 `https://jeniya.cn`，支持自定义 |
| 默认模型 | 下拉选择 | 文生图和编辑图的默认模型 |
| 默认尺寸 | 下拉选择 | 默认画布尺寸 |
| 默认数量 | Stepper | 默认生成数量（1~10） |
| 清除缓存 | 按钮 | 清除本地缓存的图片和设置 |
| 关于 / 版本号 | 文本 | 显示应用版本信息 |

## 7. 数据流

```
UI → Provider → Repository → Service → HTTP → API
                                            ↓
UI ← Provider ← Repository ← Service ← HTTP ← API
       业务模型      数据转换     原始响应
```

### 各层职责

| 层级 | 职责 | 数据格式 |
|------|------|----------|
| UI (Page) | 展示、交互 | Widget / UI State |
| Provider | 页面状态管理 | 业务模型（如 `List<ImageResult>`） |
| Repository | 数据转换、错误封装、缓存、取消 | 业务模型（`ImageResult`、`AppException`） |
| Service | HTTP 调用、基础解析 | 原始请求/响应模型 |
| API | 远端服务 | JSON |

### 详细流程

**文生图：**
1. 用户填写提示词 + 选择参数
2. 点击"生成" → Provider 设置 loading 状态 + 创建 CancelToken
3. Provider 调用 `ImageRepository.generateImage()`
4. Repository 调用 `ImageService.generateImage()` 发起 POST 请求
5. Service 返回原始响应 → Repository 解析 base64 → 转换为 `ImageResult` 列表
6. Provider 更新图片列表 → UI 自动刷新
7. 用户可点击图片进入预览页 / 保存到本地

**图编辑：**
1. 用户选择源图片（可选遮罩，支持多图）
2. 填写编辑提示词 + 选择参数
3. 点击"编辑" → Provider 设置 loading 状态
4. Provider 调用 `ImageRepository.editImage()`
5. Repository 调用 `ImageService.editImage()` 发起 multipart POST 请求
6. Service 返回原始响应 → Repository 解析 base64 → 转换为 `ImageResult` 列表
7. Provider 更新结果列表 → UI 自动刷新

**错误处理：**
- 网络错误 → Repository 捕获 → 封装为 `NetworkException` → Provider 展示 SnackBar
- API 业务错误（如 Key 无效、内容违规）→ 解析错误码 → 封装为 `ApiException` → 展示具体错误信息
- 参数验证错误 → 封装 `ValidationException` → 内联表单提示

## 8. 性能与内存优化

### 8.1 图片内存管理

| 优化点 | 方案 |
|--------|------|
| **base64 解码优化** | 在 isolate 中异步解码，避免阻塞 UI 线程 |
| **大图片采样** | GridView 展示时使用缩略图（采样缩小），点击预览再加载全尺寸 |
| **内存缓存上限** | LRU 缓存最多 20 张图片，超出自动释放，防止 OOM |
| **页面切换释放** | 切换 Tab 时释放非当前页的大尺寸图片内存，只保留缩略图 |

### 8.2 缓存策略

| 层级 | 缓存内容 | 存储方式 | 有效期 |
|------|----------|----------|--------|
| 内存缓存 | 生成的图片数据（Uint8List） | LRU Map | 会话内，上限 20 张 |
| 磁盘缓存 | 历史生成记录（提示词 + 参数 + 缩略图路径） | shared_preferences + 本地文件 | 可选，用户可清除 |

## 9. 风险与注意事项

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| **API Key 安全** | 客户端反编译可提取 Key，造成费用损失 | 提示用户妥善保管；后续可考虑加后端代理层 |
| **生成时间长** | 用户等待 10~30 秒产生焦虑 | 进度提示 + 骨架屏 + 取消按钮 |
| **大图片上传超时** | 25MB 图片上传慢、超时 | 超时时间设为 120s；提示用户压缩图片 |
| **API 费用** | 误操作 / 批量生成消耗大量额度 | 生成数量默认 1；可选生成前确认弹窗 |
| **内容合规** | 生成违规图片导致账号风险 | 使用 `moderation` 参数；用户提示协议 |
| **Web 端 CORS** | 浏览器跨域限制 | 提示用户 API 需支持 CORS，或通过代理访问 |
