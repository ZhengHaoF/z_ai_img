# 2025-07-05 工作日志

## 图片生成/编辑覆盖问题修复
- 问题：生成新图片时，已生成图片会被覆盖
- 原因：`GenerateNotifier.generate()` / `EditNotifier.edit()` 成功时直接将 `state.images` 替换为新结果
- 方案：改为追加模式
- 修改：
  - `lib/providers/generate_provider.dart` 第 164-167 行
  - `lib/providers/edit_provider.dart` 第 214-217 行
  - 将 `images: results` 改为 `images: [...state.images, ...results]`
- 效果：多次生成/编辑时，历史图片会保留在列表中，不会丢失

## flutter_background_service Release 闪退修复
- 根因：`onStart` 回调是类内静态方法，Release 模式 AOT 树摇后找不到入口点 → native crash
- 修复方式：
  - `lib/utils/background_service_helper.dart`：将 `onStart` 和 `onIosBackground` 从类内静态方法拆为顶层函数
  - `android/app/proguard-rules.pro`：新增，添加 keep 规则防止插件类被 R8 strip
  - `android/app/build.gradle.kts`：添加 `proguardFiles` 引用
  - `lib/providers/generate_provider.dart`：重新启用 `BackgroundServiceHelper.startService()`
  - `lib/providers/edit_provider.dart`：重新启用 `BackgroundServiceHelper.startService()`
- 注意：`flutter_local_notifications` 的完成/错误通知暂未启用，后续可单独修复
